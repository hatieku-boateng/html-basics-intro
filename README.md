# HTML Basics — Introductory Frontend Project

A clean, professionally styled introductory HTML & CSS webpage built as part of the **1 Million Coders Programme**. This project covers the fundamental building blocks of web development and serves as a reference guide for HTML beginners.

## What's Covered

### HTML Concepts
- Proper document structure (`<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`)
- Semantic elements — `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`
- Meta tags — charset, viewport, description, keywords
- Headings `<h1>` through `<h6>`
- Paragraphs, emphasis (`<em>`), bold (`<strong>`)
- Ordered and unordered lists
- Images with `alt` attributes
- Hyperlinks with `target="_blank"` and `rel="noopener noreferrer"`
- HTML tables — `<thead>`, `<tbody>`, `<tr>`, `<th>`, `<td>`
- Inline code with `<code>`
- HTML entities (`&amp;`, `&copy;`, `&lt;`, `&gt;`)
- HTML comments

### CSS Concepts
- CSS custom properties (variables) via `:root`
- Google Fonts integration (`Inter`, `Fira Code`)
- CSS reset and box model
- Flexbox layout
- Linear and radial gradients
- CSS pseudo-elements (`::before`, `::after`)
- Box shadows and border radius
- Hover transitions and animations
- Responsive design with `@media` queries
- Sticky navigation bar
- Zebra-striped table rows

## Getting Started

### Prerequisites
- A modern web browser (Chrome, Firefox, Edge)
- A code editor (e.g. [VS Code](https://code.visualstudio.com/))

### Run Locally

1. Clone the repository:
   ```bash
   git clone https://github.com/hatieku-boateng/html-basics-intro.git
   ```

2. Navigate into the folder:
   ```bash
   cd html-basics-intro
   ```

3. Open `1.html` in your browser — no server or build step needed.

---

## Key Learnings

- Every HTML page should start with `<!DOCTYPE html>` to enable standards mode in browsers
- The `<head>` contains metadata (invisible to users) while `<body>` contains visible content
- The `lang` attribute on `<html>` improves accessibility and SEO
- Semantic HTML elements help screen readers, search engines, and developers understand page structure
- CSS variables make it easy to maintain consistent colours and styles across a project
- The `rel="noopener noreferrer"` attribute on external links is a security best practice
