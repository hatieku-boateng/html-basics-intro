Network Models

The OSI and TCP/IP models define how communication is broken into layers — each with a specific job.

The OSI Model

The Open Systems Interconnection (OSI) model is a conceptual framework developed by ISO that divides network communication into 7 layers. Each layer has a specific role and communicates only with the layers directly above and below it.

Click each layer to expand its details.

Role: The interface between the network and end-user software. Provides services like file transfer, email, and web browsing directly to applications.

Protocols: HTTP, HTTPS, FTP, SMTP, DNS, POP3, IMAP, DHCP

Example: Your browser makes an HTTP request to load a webpage.

Role: Translates, encrypts, and compresses data so the application layer can understand it regardless of how the other system stores it.

Protocols/Formats: SSL/TLS (encryption), JPEG, PNG, MPEG, ASCII, Unicode

Example: SSL encrypts your password before it is sent over the network.

Role: Establishes, manages, and terminates communication sessions between applications. Handles authentication and reconnection after interruption.

Protocols: NetBIOS, RPC, PPTP, SIP

Example: A video call maintains a session for the duration of the call.

Role: Provides end-to-end data delivery, error recovery, and flow control. Segments large messages and reassembles them at the destination.

Protocols: TCP (reliable, connection-oriented), UDP (fast, connectionless)

Example: TCP ensures every packet of a file download arrives correctly and in order.

Role: Handles logical addressing (IP addresses) and routing — determining the best path for data to travel across multiple networks.

Protocols: IP (IPv4, IPv6), ICMP, OSPF, BGP, RIP

Example: A router examines IP addresses and forwards packets toward their destination.

Role: Provides node-to-node data transfer between directly connected devices. Uses MAC addresses to identify devices on the same network segment. Handles error detection for physical transmission.

Protocols: Ethernet (IEEE 802.3), Wi-Fi (IEEE 802.11), ARP, PPP

Example: A network switch reads MAC addresses to forward frames to the correct device.

Role: Transmits raw binary bits (0s and 1s) over a physical medium. Defines electrical signals, cable types, pin layouts, and data rates.

Examples: Ethernet cables, fibre optic, radio waves, hubs, repeaters, network interface cards (NICs)

Example: A network cable physically carries electrical pulses between a PC and a switch.

The TCP/IP Model

The TCP/IP model (also called the Internet Model) is the practical framework that actually powers the internet. It condenses the 7 OSI layers into 4 layers.

4. Application Layer

Combines OSI layers 5, 6, and 7. Handles all high-level protocols directly used by applications.

Protocols: HTTP, FTP, DNS, SMTP, DHCP, SSH

3. Transport Layer

Same as OSI Layer 4. Manages end-to-end communication and data integrity.

Protocols: TCP, UDP

2. Internet Layer

Corresponds to OSI Layer 3. Handles IP addressing and routing across networks.

Protocols: IP, ICMP, ARP, OSPF

1. Network Access Layer

Combines OSI layers 1 and 2. Handles physical transmission and link-level framing.

Examples: Ethernet, Wi-Fi, fibre optic, DSL

OSI vs TCP/IP — Comparison

Feature OSI Model TCP/IP Model
Layers 7 4
Developed by ISO (International Organization for Standardization) U.S. Department of Defense (DARPA)
Purpose Conceptual/theoretical reference model Practical implementation model for the internet
Protocol independence Generic — not tied to specific protocols Based on TCP and IP specifically
Presentation & Session layers Separate dedicated layers Merged into the Application layer
Usage today Teaching and troubleshooting reference Actual internet communication standard

TCP vs UDP

The two main Transport layer protocols serve different needs:

TCP — Transmission Control Protocol

Type: Connection-oriented
Reliability: Guaranteed delivery, ordering, error checking
Speed: Slower (acknowledgements required)
Use cases: Web browsing (HTTP), email, file transfers, online banking

UDP — User Datagram Protocol

Type: Connectionless
Reliability: No guaranteed delivery or ordering
Speed: Faster (no handshake overhead)
Use cases: Video streaming, VoIP, online gaming, DNS lookups
