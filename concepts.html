Core Concepts

Everything you need to understand how HTML works — elements, attributes, structure, and headings.

Elements & Tags

An HTML element consists of an opening tag, content, and a closing tag. Together they define what the content is and how it should be treated by the browser.

<p>This is a paragraph.</p>

Some elements are self-closing — they have no content and need no closing tag:

<img src="photo.jpg" alt="A photo" />
<br />
<input type="text" />

Block vs Inline Elements

Block Elements

Start on a new line and take up the full width. Examples: <div>, <p>, <h1>, <section>

Inline Elements

Flow within text and only take up as much width as needed. Examples: <span>, <a>, <strong>, <em>

Attributes

Attributes provide extra information about an element. They always go inside the opening tag and come in name/value pairs.

<a href="https://example.com" target="_blank" rel="noopener noreferrer">Visit</a>

Common Attributes

Attribute Used On Purpose
id Any element Unique identifier for the element
class Any element Groups elements for CSS/JS targeting
href <a> The URL the link points to
src <img>, <script> Path to an external resource
alt <img> Text description of the image
style Any element Inline CSS styling
target <a> Where to open the link (_blank = new tab)

Page Structure

Every valid HTML document follows this standard structure. Each part has a specific role:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello World</h1>
  </body>
</html>

Semantic Elements

HTML5 introduced semantic elements — tags that describe the meaning of their content, not just how it looks:

<header>

Top section of a page or article

<nav>

Navigation links

<main>

Primary content of the page

<section>

Thematic grouping of content

<article>

Self-contained piece of content

<footer>

Bottom section of a page

Headings

HTML provides 6 heading levels. They create a document hierarchy and are important for SEO and accessibility. Use them in order — don't skip levels.
