Core Concepts

The fundamental ideas that underpin all data communication systems — from signals to protocols.

Transmission Modes

The transmission mode defines the direction in which data flows between two devices.

Simplex

Data flows in one direction only. The sender can only send; the receiver can only receive.

Examples: keyboard to CPU, television broadcast, radio.

Half-Duplex

Data flows in both directions, but not at the same time. One party must stop transmitting before the other can respond.

Examples: walkie-talkies, CB radio, older network hubs.

Full-Duplex

Data flows in both directions simultaneously. Both parties can send and receive at the same time.

Examples: telephone calls, video calls, modern Ethernet, Wi-Fi.

Signal Types

Data is transmitted as either analog or digital signals.

Analog Signals

Continuous, smooth waveforms that vary in amplitude and frequency. Susceptible to noise but can carry information over long distances without repeaters.

Examples: sound waves, AM/FM radio, old telephone systems.

Digital Signals

Discrete, on/off pulses representing binary values (0 and 1). More resistant to noise, easier to process, and suited to computer systems.

Examples: computer data, fibre optic transmission, Ethernet.

Key Signal Properties

Property Definition Unit
Amplitude The strength or height of a signal wave Volts (V)
Frequency The number of wave cycles per second Hertz (Hz)
Bandwidth The range of frequencies a channel can carry; determines maximum data rate Hz or bps
Data Rate The speed at which bits are transmitted bps, Mbps, Gbps
Latency Delay between sending and receiving data Milliseconds (ms)

Transmission Media

The medium is the physical path through which a signal travels from sender to receiver.

Guided (Wired) Media

Medium How it Works Speed Use Case
Twisted Pair (UTP/STP) Pairs of copper wires twisted to reduce interference Up to 10 Gbps LAN (Cat5e, Cat6), telephone lines
Coaxial Cable Central copper conductor surrounded by insulation and a shield Up to 1 Gbps Cable TV, cable internet
Fibre Optic Transmits data as pulses of light through a glass/plastic core 100+ Gbps Internet backbone, data centres, ISPs

Unguided (Wireless) Media

Medium Frequency Range Range Use Case
Radio Waves 3 Hz – 3 GHz Very long AM/FM radio, WiFi, Bluetooth, cellular
Microwave 1 – 300 GHz Line of sight Satellite communication, point-to-point links
Infrared 300 GHz – 400 THz Very short TV remotes, short-range device communication

Protocols & Error Detection

A protocol is a set of agreed rules that govern how data is formatted, transmitted, received, and acknowledged between devices.

Why Protocols Matter

Syntax

Defines the format and structure of the data — what each field means and in what order.

Semantics

Defines the meaning of each section of data and what action should be taken upon receiving it.

Timing

Controls when data is sent, how fast, and how to handle the case where the sender is faster than the receiver.

Error Detection Methods

Method How it Works Strength
Parity Bit Appends a single bit to make the count of 1s even or odd Detects single-bit errors only
Checksum Sends the sum of all data segments; receiver recalculates and compares Moderate — used in TCP/IP
CRC (Cyclic Redundancy Check) Divides data by a polynomial; the remainder is sent as a check value Very strong — detects burst errors
